# Requirements: v1.6 영상 스토리보드

**Defined:** 2026-07-21
**Core Value:** 트래킹 장비 없이도 은하수 사진이 "잘 나오게" 하는 전 과정을 초보자가 따라 성공하게 한다 — v1.6은 명소·주제별 영상 제작 스토리보드로 "이 명소/주제에서 이런 컷을 이 순서·설정으로 담아 한 편을 만든다"는 구체 촬영·편집 계획을 독자에게 준다.

**핵심 정직성 원칙(승계):** 스토리보드 컷 썸네일은 실촬영본이 아니므로 "기획/콘셉트" 라벨 + "실촬영·완성 영상은 트립(8/13) 후 교체" 명시 / 스토리보드 장비 표기(Mini 4 Pro·RS 3 Mini·iPhone 15 Pro)는 책 기준(Mini 5 Pro·R7)과 대조해 수치 재확인·미확인 표기(단정 금지) / 각 트랙 촬영·편집법은 재설명 없이 기존 1·2·3부 링크로 승계(중복 금지) / 이미지는 ~2000px JPEG 리사이즈로만 삽입, 자체 영상 임베드 금지.

## v1 Requirements

### 영상 스토리보드 반영 (SB)

- [ ] **SB-01**: 독자가 2부 "명소별 영상 스토리보드" 신규 챕터에서 **드론 영상 스토리보드 3종**(차강소브라·몽고말떼·어르헝)을 촬영 계획(샷 리스트·촬영 설정·동선·편집 흐름·BGM)으로 본다. 어르헝은 "보너스/추가 명소"로 표기(코스 편입 아님).
- [ ] **SB-02**: 독자가 **드론+지상 통합 스토리보드 3종**(홍고린엘스·욜링암·바가가즈링촐로)을 "세 카메라 통합 촬영 계획"으로 보고, 4부 하루 현장 운용(field-day)의 세 카메라 오케스트레이션 실제 사례로 연결된다.
- [ ] **SB-03**: 독자가 **은하수 타임랩스 영상 스토리보드**(몽골밤하늘, Canon R7)를 3부에서 촬영·편집 계획으로 본다(3부 배치 또는 교차링크).

### 정직성·자산 처리 (ASSET)

- [ ] **ASSET-01**: 모든 스토리보드 이미지가 **기획/콘셉트로 정직하게 라벨**되고(완성 영상 예시 아님·실촬영/완성 영상은 트립 8/13 후 교체), **~2000px JPEG로 리사이즈**되어 `src/images/`에 삽입된다(자체 영상 임베드 없음).
- [ ] **ASSET-02**: 각 스토리보드의 **장비·설정 표기가 책 기준(Mini 5 Pro·Canon R7)과 대조**되어, 불일치(Mini 4 Pro 등)는 "재확인/미확인" 또는 "5 Pro 기준 재검증 필요"로 정직하게 조정된다(수치 단정 금지).

### 발견성·연결 (LINK)

- [ ] **LINK-01**: 독자가 각 **명소 페이지(1·2·3부)에서 해당 스토리보드로**, **4부 field-day에서 통합 스토리보드 3종으로** 이동할 수 있고, 스토리보드 페이지들은 촬영·편집법을 기존 파트로 링크 승계하며, **책 전체 내부 링크 dead 0**을 유지한다(최종 게이트).

## v2 Requirements

- 트립(8/13) 후 저자 실촬영본·완성 영상 프레임으로 스토리보드 정지프레임 교체, 실제 편집본 링크
- 바양작(Bayanzag) 스토리보드 신설(현재 미보유 → 보류)
- 어르헝(오르혼)을 코스에 정식 편입할지 결정(현재는 보너스 명소)

## Out of Scope

| Feature | Reason |
|---------|--------|
| 각 트랙 촬영·편집법 재설명 | 이미 1·2·3부에 있음 — 스토리보드 페이지는 링크로 승계, 중복 금지 |
| 자체 영상 파일 임베드/호스팅 | GitHub Pages 용량 한계 — 스토리보드 이미지(리사이즈)+텍스트로만 |
| 스토리보드를 "완성 영상 예시"로 소개 | 콘셉트/기획 이미지·실촬영본 부재 — 정직성(계획 도구로만) |
| 바양작 스토리보드 | 자료 미보유 → v2 보류 |
| 어르헝 코스 편입 | 비고비·현 일정 밖 → 보너스 명소로만 |
| Mini 4 Pro 설정값을 5 Pro로 단정 변환 | 미검증 → 재확인/미확인 표기, 단정 금지 |
