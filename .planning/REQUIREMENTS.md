# Requirements: v1.5 현장 워크플로

**Defined:** 2026-07-20
**Core Value:** 트래킹 장비 없이도 은하수 사진이 "잘 나오게" 하는 전 과정을 초보자가 따라 성공하게 한다 — v1.5는 세 트랙(여행·드론·은하수)을 하나의 여행으로 잇는 현장 워크플로를 신설.

**핵심 정직성 원칙(승계):** 하루 타임라인 시각은 예시 템플릿("날짜·계절·일몰시각별 재확인") / 장비 무게·카드 용량·파워뱅크 용량·요금은 검증 시만 단정, 아니면 "미확인" / 특정 백업 장비·인버터·클라우드는 추천만(구매 강요·정밀 스펙 단정 금지) / 실제 촬영본·화면 캡처·하루 실사례는 트립(8/13) 후 교체 / 각 트랙 촬영·편집법은 재설명 없이 기존 파트 링크로 승계.

## v1 Requirements

### 여행 전 준비 (PREP)

- [ ] **PREP-01**: 독자가 세 장비(R7·드론·야간 촬영)를 한 번에 챙기는 **통합 패킹·준비물 리스트**로 빠짐없이 준비한다(부록 체크리스트를 허브로 링크, 중복 신설 없이).
- [ ] **PREP-02**: 독자가 출발 전 **촬영 계획**(신월·달 위상·일정별 명소와 낮/밤 배분)과 **앱·지도 오프라인 준비**(PhotoPills·Stellarium·오프라인 지도 다운로드)를 마친다.

### 하루 현장 운용 (FIELD)

- [ ] **FIELD-01**: 독자가 도착부터 밤까지의 **하루 타임라인**(낮 여행 사진 → 오후 드론 → 골든/블루아워 → 천문박명 후 은하수·스타트레일 → 백업)을 예시 시간표와 함께 따라 한다.
- [ ] **FIELD-02**: 독자가 **세 카메라(R7·드론·야간 R7) 오케스트레이션** — 언제 무엇을 꺼내 쓰나, 우선순위, 셔터 찬스를 놓치지 않는 법 — 을 이해하고 하루 안에서 전환한다.

### 귀국 후 정리 (POST)

- [ ] **POST-01**: 독자가 귀국 후 촬영본을 **오프로드·백업(3-2-1 원칙)**하고 안전하게 보관·아카이브한다.
- [ ] **POST-02**: 독자가 대량 촬영본을 **선별(컬링)**하고, **트랙별 현상·편집 순서**(여행=Lightroom / 드론 영상=CapCut / 은하수=스태킹)로 완성한다(기존 편집 파트를 순서로 잇는 허브).

### 데이터·전원 운용 (DATA)

- [ ] **DATA-01**: 독자가 **카드 전략**(용량·개수·로테이션·현장 포맷 규칙)과 **현장 RAW 백업** 방법(노트북/휴대 SSD/폰 오프로드)을 안다.
- [ ] **DATA-02**: 독자가 **멀티데이 오프그리드 전원**(장비별 배터리·파워뱅크·차량 인버터/시가잭 충전·저온 배터리 대응·충전 우선순위)을 운용한다.

## v2 Requirements

- 트립(8/13) 후 저자의 **실제 하루 사례**·현장 사진·백업 스크린샷으로 교체
- 영어판 / 인쇄용 PDF / 상단 부(部) 네비바

## Out of Scope

| Feature | Reason |
|---------|--------|
| 각 트랙 촬영·편집법 재설명 | 이미 1·2·3부에 있음 — 링크로 승계, 중복 금지 |
| 특정 백업 장비·인버터·SSD 모델 추천/정밀 스펙 | "추천만" 원칙·미확인 방지, 유형·기준만 제시 |
| 클라우드 백업 서비스 상세 요금 비교 | 요금 변동·정직성, 원칙(3-2-1)만 |
| 실제 촬영본·화면 캡처·하루 실사례 임베드 | 트립(8/13) 후에만 정직하게 채움 |
| 하루 타임라인의 고정 시각 단정 | 날짜·계절·일몰시각 의존 — 예시 템플릿으로만 |
