사용 원본(둘 다 Bernard Gagnon, CC0, Wikimedia — 재호스팅 무조건 허용):
- 대낮 강광(레시피 a): `src/images/travel-gallery/khongoryn-els.jpg` (밝은 사막 사구)

정직한 전/후 만들기(image_facts 규약):
- **after = CC0 원본을 재최적화**(최대 2000px, 메타 제거):
  ```
  magick src/images/travel-gallery/khongoryn-els.jpg -resize '2000x2000>' -strip -quality 82 src/images/editing/recipe-midday-after.jpg
  magick src/images/travel-gallery/bayanzag.jpg      -resize '2000x2000>' -strip -quality 82 src/images/editing/recipe-goldenhour-after.jpg
  ```
- **before = 같은 CC0 원본에서 의도적으로 '플랫하게' 만든 버전**(대비↓+채도↓+약간의 노출/WB 이동 → 미현상 플랫 캡처를 모사). 레시피별로 톤을 튜닝:
  - 대낮(하이라이트가 날 것 같고 밋밋한 느낌): 대비/채도 낮추고 살짝 밝게(플랫)
    ```
    magick src/images/travel-gallery/khongoryn-els.jpg -resize '2000x2000>' -strip -brightness-contrast -6x-22 -modulate 100,72,100 -quality 82 src/images/editing/recipe-midday-before.jpg
    ```
  - 골든아워(따뜻함이 빠진 차갑고 밋밋한 플랫): 채도 더 낮추고 색온도를 차갑게(파랑 쪽) 살짝 이동
    ```
    magick src/images/travel-gallery/bayanzag.jpg -resize '2000x2000>' -strip -brightness-contrast -4x-24 -modulate 100,64,100 -channel R -evaluate multiply 0.94 +channel -channel B -evaluate multiply 1.06 +channel -quality 82 src/images/editing/recipe-goldenhour-before.jpg
    ```
  값은 결과를 보고 조정 가능(정직한 '플랫' 인상이 목표이지 과장 아님). 4개 파일 모두 2000px 이하·-strip·품질 82 내외.

**정직성(비협상):** 이 쌍은 진짜 RAW 현상 결과가 아니다. before는 인위적으로 플랫하게 만든 파생물이며 캡션에서 반드시 그렇게 밝힐 것(캡션은 Task 2에서 작성). 여기서는 파일만 생성.

스텁을 완전히 대체. 문체는 지침체 존댓말·괄호 뜻풀이(index.md/camera-settings.md 스타일). Adobe 기능 이름은 FEATURES.md 검증 표기만. 방향은 "이쪽으로 미세요" 상대 안내이지 고정값이 아님을 인트로에 명시(FEATURES.md 주의).

**인트로:** 같은 17단계 순서라도 장면마다 방향이 다름을 짧게, 앞 장 [필수 보정 순서](develop-order.md)를 전제로 링크. 값이 아니라 **이유(reasoning)**를 배우는 것이 목표.

**레시피 a~e — 각각 소제목 + "어떤 슬라이더 / 어느 방향 / 왜" 목록.** FEATURES.md "Travel-Specific Recipes (a)~(e)" 근거:
- **(a) 대낮 강광 사막(날린 하이라이트·강한 그림자·고대비):** Highlights −−(하늘/모래 복원), Shadows +(그늘 열기), Whites − 약간, Blacks 유지 / Contrast − 약간 / Dehaze + 소·Clarity + 소(정오 헤이즈) / Vibrance +, Saturation 중립, WB Temp − 종종(오렌지 모래 회피) / 선택: 상단 Linear Gradient로 Exposure − 하여 하늘 잡기. **왜**: 정오의 거대한 동적범위를 출력 가능한 톤으로 압축.
- **(b) 골든아워/황혼 따뜻한 장면:** WB 따뜻함 **유지**(중화 금지 — 따뜻함이 주제), Temp 절제 / Shadows +(전경이 검게 죽지 않게), Highlights −(태양 글로우 유지) / Color Grading: 따뜻한 하이라이트 + 살짝 차가운 섀도우, HSL 오렌지/레드 Luminance·Saturation 소폭↑ / Dehaze + 아주 소, 섀도우 들면 노이즈 → Denoise 고려(Phase 18). **왜**: 황혼의 따뜻함 자체가 피사체.
- **(c) 역광/고대비(실루엣 위험):** 전역으로 Shadows ++/Blacks +로 어두운 피사체 구제 + Highlights −−로 밝은 림/하늘 진정, **더 나은 방법**: 마스크 **Select Subject → Exposure +** 국소로(전체를 흐리지 않게). Clarity/Dehaze 과하면 헤일로 → 낮게. 의도 선택(실루엣 유지 vs 피사체 드러내기)을 **선택으로** 가르침. **왜**: 국소 리프트가 프레임 전체를 탁하게 만들지 않음.
- **(d) 인물/피부톤(유목민·현장 스냅):** WB 살짝 따뜻(Tint 녹/마젠타 캐스트 점검) / **Vibrance(채도 아님)** — Saturation은 피부를 오렌지로 / HSL 오렌지 Luminance + 소(피부 밝히기), 과하면 Saturation 오렌지 − / 마스크 **Select People** → 피부에서 Texture/Clarity 소폭↓, Exposure + 로 얼굴 리프트 / 피부에 Clarity는 낮게/음수(양수는 얼굴을 늙고 거칠게). **왜**: 피부는 눈이 기준 삼는 색.
- **(e) 살짝 날린 하늘 복원:** 먼저 히스토그램/Alt-drag로 Highlights+Whites 클리핑 확인 → Highlights −−, Whites −. RAW는 채널당 ~1스톱 복원 가능, JPEG은 불가(RAW 촬영 이득). **완전히 클리핑(순백·데이터 없음)이면 복구 불가** → 마스크 **Select Sky → Exposure/Highlights 낮춤**, Temp/Tint·Saturation로 톤을 **시늉**하거나 수용. **정직성**: 날린 것은 사라진 것. 예방 콜백(브라케팅+HDR, ETTR, 그라디언트 ND)로 촬영으로 연결.

각 마스킹 언급(c=Select Subject, d=Select People, e=Select Sky)은 **도구명만 짧게 + [국소 보정 — 마스킹·디테일](masking-and-detail.md)(Phase 18)로 링크**. 마스킹 상세는 여기서 금지.

**과보정(P3) 경고 박스(필수, blockquote `>` 형식):** PITFALLS Pitfall 3 근거로 대표 증상 나열 — Clarity/Texture/Dehaze 과다=가장자리 **헤일로**·거친 질감, Saturation/Vibrance 과다=형광 하늘·오렌지 피부/모래, Blacks 과다=크러시드 블랙(디테일 소실), Dehaze로 하늘 과다=밴딩·부자연 청록. **예방**: 슬라이더 넣고 한 발 물러서기(값 절반), 100%↔축소뷰 확인, before/after 토글로 원본 비교, "덜 한 것이 낫다". 본문에 "과보정" 또는 "P3" 개념이 박스로 분명히 존재해야 함(grep 대상). 이 박스는 17-02 전용(develop-order.md는 한 줄 주의만).

**촬영으로 예방 콜백(각 레시피 또는 별도 절):** 편집으로 메우기보다 촬영에서 예방하는 링크 —
