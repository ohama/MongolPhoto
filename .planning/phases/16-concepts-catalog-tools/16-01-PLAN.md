1. **비파괴 편집이란(EDIT-03):** 원본 CR3는 픽셀 한 개도 바뀌지 않고, 슬라이더 조작은 "편집 지시서"로만 저장된다. Lightroom Classic은 이 지시서를 **카탈로그(.lrcat 파일 — 원본 경로·미리보기·편집 내역을 담는 데이터베이스)**에 보관한다는 점을 설명. 원본만 살아 있으면 편집은 언제든 재현·되돌리기 가능함을 강조.

2. **가져오기(Import)와 카탈로그의 관계:** LR은 "폴더 뷰어"가 아니라 카탈로그가 원본 파일의 **위치를 참조**하는 구조임을 설명. 가져오기 시 폴더 구조를 먼저 정하고 이후 건드리지 않는 습관(PITFALLS P1 예방)을 안내.

3. **현상 모듈(Develop) 개요 — 글/표로:** 현상 모듈이 보정이 일어나는 곳이며, 위에서 아래로(전역→국소) 진행하는 패널 흐름이 있음을 **한 문단 + 짧은 표**로 소개. 세부 17단계 순서는 이 장에서 완성하지 말고 다음 파트([필수 보정 순서](develop-order.md))에서 다룬다고 예고(중복 회피). Adobe UI 스크린샷 자리에는 HTML 주석 플레이스홀더 삽입:
   `<!-- 스크린샷: 현상 모듈 패널 — 저자 직접 캡처 예정 -->`
   그리고 캡션 텍스트로 "(스크린샷: 저자 직접 캡처 예정)"을 남긴다. **Adobe UI 이미지를 절대 삽입/재호스팅하지 말 것**(저작권).

4. **⚠️ 경고 박스 1 — 카탈로그 밖에서 파일을 옮기지 마세요 (P1, Critical):** blockquote(`>`)로 강조 박스 구성. 내용: Finder/탐색기로 파일·폴더를 옮기거나 이름을 바꾸면 LR이 원본을 잃어 썸네일에 **느낌표(!)**가 뜨고 "사진 없음(missing photo)"이 된다. 예방: 이동/이름변경/정리는 **반드시 LR 내부(라이브러리)에서**. 사고 시 "폴더/사진 찾기(Find Missing)"로 재연결. (PITFALLS P1 그대로.)

5. **⚠️ 경고 박스 2 — 카탈로그 백업 ≠ 원본 백업 (P2, Critical):** blockquote 박스. 내용: 종료 시 뜨는 "카탈로그 백업"은 .lrcat(편집 내역)만 백업하며 **원본 RAW는 백업하지 않는다**. 두 가지를 따로 백업(① 원본 RAW 외장/클라우드, ② 카탈로그+프리뷰). 비파괴 편집이라 원본만 살아 있으면 재현 가능함을 재강조. 3-2-1 백업 원칙 한 줄 언급. (PITFALLS P2 그대로.)

6. **교차 링크:** RAW/WB 기초는 [RAW와 화이트밸런스](../02-fundamentals/raw-and-wb.md)로 링크(재설명 금지). 도구 선택은 [Lightroom Classic vs 다른 편집 앱](software-comparison.md)로.
