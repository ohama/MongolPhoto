<critical_facts>
- **mdBook 루트는 `src/`** (`src/book.toml`에 `src = "."`, `build-dir = "../docs"`, `create-missing = false`). SUMMARY.md는 `src/SUMMARY.md`이고 챕터 링크는 `src/` 기준 상대경로다. **모든 SUMMARY 링크는 실제 파일을 가리켜야 하며, 아니면 빌드가 실패한다.**
- **빌드 명령**: `mdbook build src`. 빌드 후 `docs/`가 재생성되며 diff가 생긴다. 실행자는 검증 후 **`docs/`를 반드시 되돌리고 커밋하지 않는다**: `git restore docs/ 2>/dev/null; git clean -fdq docs/ 2>/dev/null`. CI가 `docs/`를 다시 빌드하므로 `src/`·README·.planning 파일만 커밋한다.
- **커밋 규율**: 파일을 개별 스테이징(`git add <경로>`)한다. **`git add .` 금지, `docs/` 커밋 금지.**
- **정직성**: 스텁은 실제 보정 내용이 아니다. 제목(H1) + 한두 줄 소개 + "이 장에서 다룰 내용" 티저 목록 + "준비 중" 안내만 넣는다. 실제 Lightroom 단계·수치·레시피는 쓰지 않는다(그건 Phase 16-19).
- **문체**: 지침체 존댓말(`~습니다`/`~합니다`). `src/11-travel/index.md`의 톤을 따른다.
</critical_facts>
