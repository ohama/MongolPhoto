도입: 편집의 뼈대는 "어떤 장면을, 어떤 순서로, 얼마나 길게, 어떤 리듬으로" 보여 줄지 정하는 일이라는 1문단.

1. **가져오기(Import)** — 드론 클립·음악·소재를 프로젝트로 임포트하고, **프로젝트 해상도·기준 fps를 먼저 정한다**(예: 4K·30fps). 촬영 파트에서 정한 기준 fps와 맞추라는 **백링크** → [드론 영상 촬영 기초](video-shooting.md)의 프레임레이트/슬로모 헤드룸 결정. (음악 임포트 관련 라이선스 주의는 색보정과 음악 챕터에서 자세히 다룬다고 전방 링크 [색보정과 음악](capcut-grade-speed.md).)

2. **타임라인 배치(Arrange)** — 여러 클립을 시간축에 순서대로 늘어놓기.

3. **컷·트림(Cut / Trim)** — 각 클립에서 "좋은 구간"만 남기고 이착륙·흔들림·표류·정지 구간을 잘라냄. **목표 길이보다 살짝 길게** 남겨 두라는 편집 여유 팁(시작/끝 여유 프레임).

4. **스토리 순서(Story order)** — establishing(넓은 설정 샷) → build(전개) → climax(절정) → resolve(마무리) 흐름으로 재배열. "가장 센 샷을 아무 데나 두지 말고 리듬 안에 배치."

5. **속도(Speed)** — 두 갈래를 명확히:
   - **슬로모션** — 촬영 때 높은 fps로 찍어 둔 클립을 느리게. 부드러움을 위해 옵티컬 플로(광학 흐름 보간) 옵션. 슬로모는 **촬영 단계에서 헤드룸을 만들어 둬야** 가능하다는 점을 다시 강조(백링크 video-shooting.md).
   - **스피드 램프(speed curve, 속도 곡선)** — 한 클립 안에서 속도가 천천히 → 빠르게(또는 반대로) 변하는 것. 리빌·전환 순간에 극적. 초보는 프리셋 커브에서 시작해 포인트를 조정.
