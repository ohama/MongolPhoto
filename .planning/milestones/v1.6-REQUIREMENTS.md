**Core Value:** 트래킹 장비 없이도 은하수 사진이 "잘 나오게" 하는 전 과정을 초보자가 따라 성공하게 한다 — v1.6은 명소·주제별 영상 제작 스토리보드로 "이 명소/주제에서 이런 컷을 이 순서·설정으로 담아 한 편을 만든다"는 구체 촬영·편집 계획을 독자에게 준다.

**핵심 정직성 원칙:** 스토리보드 컷 썸네일은 실촬영본이 아니므로 "기획/콘셉트" 라벨 + "실촬영·완성 영상은 트립(8/13) 후 교체" 명시 / 장비 표기(Mini 4 Pro·RS 3 Mini·iPhone 15 Pro)는 책 기준(Mini 5 Pro·R7)과 대조해 재확인·미확인(단정 금지) / 촬영·편집법 재설명 없이 기존 1·2·3부 링크 승계 / 이미지는 ~2000px JPEG 리사이즈로만, 자체 영상 임베드 금지.

## v1 Requirements

### 영상 스토리보드 반영 (SB)

- [x] **SB-01**: 독자가 2부 "명소별 영상 스토리보드" 신규 챕터에서 드론 영상 스토리보드 2종(차강소브라·몽고말떼)을 촬영 계획(샷 리스트·설정·동선·편집·BGM)으로 본다. — Phase 30
- [x] **SB-02**: 독자가 드론+지상 통합 스토리보드 4종(홍고린엘스·욜링암·바가가즈링촐로·바양작)을 "세 카메라 통합 촬영 계획"으로 보고, 4부 field-day 세 카메라 오케스트레이션 사례로 연결된다. — Phase 30
- [x] **SB-03**: 독자가 은하수 타임랩스 영상 스토리보드(몽골밤하늘, Canon R7)를 3부에서 촬영·편집 계획으로 본다. — Phase 31 (3-astro/6-bonus/timelapse.md 섹션)

### 정직성·자산 처리 (ASSET)

- [x] **ASSET-01**: 모든 스토리보드 이미지가 기획/콘셉트로 정직하게 라벨되고, ~2000px JPEG로 리사이즈되어 `src/images/storyboards/`에 삽입된다(자체 영상 임베드 없음). — Phase 29
- [x] **ASSET-02**: 각 스토리보드의 장비·설정 표기가 책 기준(Mini 5 Pro·Canon R7)과 대조되어, 불일치(Mini 4 Pro 등)는 "재확인/미확인"으로 정직하게 조정된다(수치 단정 금지, index.md 대조표 단일화). — Phase 29(대조표)+Phase 30/31(페이지별 적용)

### 발견성·연결 (LINK)

- [x] **LINK-01**: 독자가 각 명소 페이지(1·2·3부)에서 해당 스토리보드로, 4부 field-day에서 통합 스토리보드 4종으로 이동할 수 있고, 스토리보드 페이지는 촬영·편집법을 기존 파트로 링크 승계하며, 책 전체 내부 링크 dead 0을 유지한다(최종 게이트). — Phase 30(field-day 역링크)+Phase 31(명소 15 역링크·최종 게이트, 링크 1080 REAL DEAD 0)
